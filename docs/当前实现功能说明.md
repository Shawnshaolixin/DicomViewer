# 当前实现功能说明

## 1. 文档目的

这份文档用于说明当前学习项目已经实现到什么程度，哪些功能可以直接体验，哪些仍然是骨架或占位实现。

## 2. 当前可用功能

### 2.1 工程与启动

1. 已建立基于 .NET 10 和 WPF 的解决方案。
2. 已拆分为 Wpf、Application、Domain、Infrastructure、Rendering、Shared、Tests 七个项目。
3. 程序已接入 Prism，可通过容器启动并自动绑定主窗口与 ViewModel。

### 2.2 数据导入

1. 支持输入本地目录路径并扫描目录中的 DICOM 文件。
2. 支持从 DICOM 元数据中读取患者、检查、序列、实例基础信息。
3. 支持按 Patient -> Study -> Series -> Instance 进行分组。
4. 在未输入目录时，会自动回退到内置样例数据，方便学习和演示。
5. 导入时会统计可解析实例数，并对目录不存在、无可解析文件、部分文件跳过等情况给出明确说明。

### 2.3 影像显示

1. 支持将灰度 DICOM 图像读取为可显示的 Gray8 位图。
2. 支持在中央视口显示当前序列的当前切片和当前帧。
3. 支持多帧实例的逐帧浏览。
4. 若文件不存在、像素格式不支持或当前是样例数据，则回退为占位视图，并显示失败原因。
5. 已支持 RGB 彩色 DICOM 的基础显示与缩略图预览。

### 2.4 序列与切片浏览

1. 左侧列表显示当前已加载的序列。
2. 点击序列可切换当前活动序列。
3. 支持上一张、下一张切片切换。
4. 支持上一帧、下一帧切换多帧图像。
5. 状态栏和视口叠加层会同步显示当前切片与帧信息。
6. 左侧序列列表已支持首张图像缩略图预览。

### 2.5 窗宽窗位与缩放

1. 支持软组织窗、肺窗、骨窗预设。
2. 支持 WW+、WW-、WL+、WL- 按钮调整窗宽窗位。
3. 支持在“窗宽窗位”模式下通过鼠标拖拽实时调窗。
4. 窗宽窗位变化会触发图像重新映射和重新显示。
5. 支持放大、缩小按钮与鼠标滚轮缩放当前视图。
6. 支持在“平移”模式下通过鼠标拖拽移动图像。
7. 支持左旋 90°、右旋 90°、水平翻转、垂直翻转。
8. 当前缩放、平移、旋转和翻转状态会显示在视口信息中。

### 2.6 测量与标注

1. 支持长度测量模式，可通过两点绘制距离标注。
2. 支持角度测量模式，可通过三点绘制角度标注。
3. 支持测量过程中的鼠标预览。
4. 支持清空当前序列的测量标注。
5. 支持在右侧标注列表中选中并删除单条已完成标注。

### 2.7 信息展示

1. 右侧面板显示患者信息、检查信息、当前工具模式、视口状态。
2. 顶部显示当前导入说明。
3. 底部状态栏显示导入状态、切片信息、帧信息、窗宽窗位信息。

### 2.8 测试

1. 已有基础测试覆盖样例数据加载。
2. 已有基础测试覆盖空目录导入。
3. 已有基础测试覆盖单帧灰度图像读取。
4. 已有基础测试覆盖不同窗宽窗位对像素映射结果的影响。
5. 已有基础测试覆盖多帧图像加载。
6. 已有基础测试覆盖工作区帧切换与基础测量标注。
7. 已有测试覆盖工作列表加载、联锁失败、曝光状态流转、PACS 结果处理。
8. 已有测试覆盖模拟 DICOM 文件生成和模拟曝光产物可读性。

### 2.9 曝光控制台

1. 已新增独立的曝光控制台页面，可在主窗口顶部导航进入。
2. 已支持加载模拟工作列表，并从列表中选择当前检查任务。
3. 已支持创建检查会话，并显示当前患者、检查、设备状态和流程状态。
4. 已支持编辑曝光参数，包括 kV、mA、ms、mAs、SID、BodyPart、Projection 和 AEC。
5. 已支持编辑联锁参数范围，包括 kV、mA、ms、mAs 和 SID 的最小值与最大值。
6. 已支持模拟探测器连接、球管预热、防护门关闭和 PACS 可用等联锁开关。
7. 已支持显式执行联锁检查，并展示可读失败原因。

### 2.10 采集模拟、DICOM 与 PACS

1. 已支持模拟一次曝光流程，并驱动 Ready、Exposing、Processing 等状态变化。
2. 已支持根据曝光参数生成伪灰度图像，并封装为可读取的 DICOM 文件。
3. 已写入 Patient、Study、Series、SOP Instance UID、Rows、Columns、PixelData、窗宽窗位和基础曝光参数等 DICOM 信息。
4. 已支持配置本地输出目录，模拟曝光生成的 DICOM 文件会写入当前控制台配置的目录。
5. 已支持使用 C-ECHO 验证 Orthanc/PACS 连通性。
6. 已支持使用 C-STORE 将生成的 DICOM 文件发送到 PACS，并反馈成功或失败结果。
7. 已支持记录工作列表加载、参数修改、联锁检查、曝光执行、DICOM 生成、PACS 验证与发送等审计日志。
8. 已支持将 PACS 配置和联锁参数范围持久化到本地 SQLite 数据库，重启应用后自动恢复。
9. 已支持将审计日志写入本地 SQLite 数据库，并在控制台中回读显示。
10. 已支持将检查会话状态、最近一次曝光结果和 PACS 发送记录写入本地 SQLite 数据库。
11. 已支持配置 Orthanc REST 端口，用于第二阶段的远端查询与回取。
12. 已支持配置远端查询筛选条件，包括患者姓名、患者 ID、检查描述、Modality 和检查日期范围。
13. 已支持配置本地 DICOM 接收 Host / Port，用于标准 C-MOVE 回取时接收 Orthanc 推送的实例。

### 2.11 回看闭环

1. 已支持在控制台中通过“回看结果”按钮直接回看本次曝光生成的 DICOM 数据，无需先发送 PACS。
2. 已支持在 PACS 发送成功后自动跳转回查看器，并导入生成文件所在目录。
3. 已复用现有查看器完成回看，因此仍可使用已有的窗宽窗位、缩放、平移、旋转和基础测量能力。
4. 已支持在控制台中显示最近历史检查列表，并可从历史记录直接回看已有结果。
5. 已支持在控制台中查询 Orthanc 最近检查，并将选中检查回取到本地目录后直接跳转回查看器。
6. 已支持在控制台中通过标准 DICOM C-FIND 查询远端检查，并通过 C-MOVE 回取到本地接收端后直接跳转回查看器。

## 3. 当前限制

1. 当前只支持单样本灰度图，不支持彩色 DICOM。
2. 当前测量标注尚未支持拖拽修改与画布内直接编辑。
3. 当前未实现缩略图预览。
4. 当前 PACS 集成已具备 C-ECHO / C-STORE、基于 Orthanc REST 的 Query / Retrieve，以及标准 DICOM C-FIND / C-MOVE 的最小学习闭环，但未覆盖 MWL、MPPS 和更完整的异常恢复流程。
5. 当前历史检查列表已集成在控制台页面中，现已支持关键字筛选和分页，但尚未提供独立历史页面。
6. 当前未实现 MPR、多视口联动等高级阅片能力。
7. 当前 MPPS 已支持 Started / Completed / Discontinued 的最小上报闭环，但尚未覆盖更完整的执行步骤明细和编码化中止原因。

## 4. 适合作为学习入口的代码位置

1. 工作区状态汇总与业务编排：src/DicomViewer.Application/Services/WorkspaceService.cs
2. DICOM 元数据导入：src/DicomViewer.Infrastructure/Data/FileSystemStudyCatalogService.cs
3. DICOM 像素到 Gray8 的转换：src/DicomViewer.Infrastructure/Imaging/DicomViewportImageService.cs
4. WPF 主界面命令与 Prism 绑定：src/DicomViewer.Wpf/ViewModels/ShellViewModel.cs
5. 主界面布局：src/DicomViewer.Wpf/MainWindow.xaml
6. 曝光控制台工作流编排：src/DicomViewer.Application/Services/ExamWorkflowService.cs
7. 曝光控制台界面与交互：src/DicomViewer.Wpf/ViewModels/ExposureConsoleViewModel.cs
8. 模拟 DICOM 生成：src/DicomViewer.Infrastructure/Dicom/SimulatedDicomBuilder.cs
9. Orthanc / PACS 发送与回取：src/DicomViewer.Infrastructure/Pacs/OrthancStoreService.cs

## 5. 建议的下一步学习顺序

1. 先读 WorkspaceService，理解 UI 和 Application 层如何通过快照通信。
2. 再读 FileSystemStudyCatalogService，理解 DICOM 文件如何分组为领域模型。
3. 再读 DicomViewportImageService，理解窗宽窗位如何映射到 8 位灰度图。
4. 再读 ExamWorkflowService 和 DefaultInterlockService，理解工作列表、联锁、状态机和审计如何编排。
5. 再读 SimulatedDicomBuilder 和 OrthancStoreService，理解模拟采集结果如何转成 DICOM 并发送到 PACS。
6. 最后结合 App.xaml.cs、ExposureConsoleViewModel、ViewerWorkspaceViewModel 和 MainWindow.xaml，看 Prism 容器、页面导航和回看闭环如何串起来。
