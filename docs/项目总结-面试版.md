# 项目总结（面试版）

## 1. 项目定位

这是一个面向学习的 WPF 医学影像项目，目标不是一次性做成完整商用系统，而是围绕“DICOM 查看 + 曝光流程模拟 + PACS/工作流闭环”搭建一条可运行、可扩展、可讲清楚的主链路。

这个项目的价值不只是“把图显示出来”，而是帮助作者从传统 WPF 客户端开发，过渡到医疗影像软件开发的基础认知：

1. 理解 DICOM 的数据组织方式。
2. 理解医学影像查看器的基础显示能力。
3. 理解检查工作流、PACS 通信、MWL、MPPS 的位置。
4. 理解医疗软件为什么更强调稳定性、审计和可追踪性。

## 2. 这个项目已经覆盖了什么

### 2.1 客户端与架构

1. 使用 WPF + Prism 组织桌面端界面。
2. 按 Wpf、Application、Domain、Infrastructure、Rendering、Shared、Tests 分层。
3. 由 Application 层统一编排业务状态，避免 ViewModel 直接耦合 DICOM 解析和网络通信。

### 2.2 DICOM 数据与图像查看

1. 支持扫描本地目录并读取 DICOM 元数据。
2. 支持按 Patient -> Study -> Series -> Instance 分组。
3. 支持灰度图与基础 RGB 图像显示。
4. 支持窗宽窗位、缩放、平移、旋转、翻转和基础测量。

### 2.3 医疗工作流学习

1. 支持模拟工作列表加载。
2. 支持检查会话创建、联锁检查和曝光状态流转。
3. 支持模拟生成 DICOM 文件。
4. 支持 C-ECHO、C-STORE、C-FIND、C-MOVE 的最小学习闭环。
5. 支持 MPPS Started / Completed / Discontinued 的最小上报闭环。

### 2.4 工程能力

1. 使用 SQLite 持久化配置、审计日志、检查会话和 PACS 发送记录。
2. 对关键流程保留审计信息，便于排查和回溯。
3. 已有基础自动化测试覆盖主链路。

## 3. 这类项目在面试里最能体现什么

### 3.1 不只是会写界面

面试官通常不只想看你会不会写 WPF 页面，而是想看你是否能把界面层、业务状态、图像处理、网络通信、持久化和异常处理串成完整系统。

### 3.2 不只是会调用 fo-dicom

真正的考察点通常是：

1. 你是否理解 Patient / Study / Series / Instance 的层次。
2. 你是否理解窗宽窗位、切片、多帧等基础显示概念。
3. 你是否知道 PACS、MWL、MPPS 在工作流中的作用。
4. 你是否知道医疗软件为什么需要审计、重试、状态一致性和错误追踪。

### 3.3 是否具备工程落地意识

医疗软件开发里，很多功能“能跑”还不够，还要考虑：

1. 状态是否可恢复。
2. 日志是否可追踪。
3. 失败后是否可补发或重试。
4. 界面展示和底层服务是否解耦。

## 4. 面试时可以重点讲的 6 个主题

### 4.1 我如何做分层

可以重点说明：

1. WPF 层负责绑定、命令和视图交互。
2. Application 层负责工作区和检查流程编排。
3. Infrastructure 层负责 DICOM、PACS、SQLite 等外部能力封装。
4. 这样做的好处是界面、业务和外部库解耦，便于测试和扩展。

### 4.2 我如何做本地目录导入与查看

可以重点说明：

1. 扫描目录后读取 DICOM 元数据。
2. 按患者、检查、序列、实例分组。
3. 由工作区服务汇总成适合界面展示的快照。
4. 界面只消费快照，不直接处理底层解析细节。

### 4.3 我如何理解图像显示

可以重点说明：

1. 窗宽窗位本质上是像素值映射。
2. 缩放、平移、旋转和测量都涉及视口状态和坐标变换。
3. 图像显示不仅是读取像素，还要处理失败回退、占位图和状态提示。

### 4.4 我如何理解曝光控制台与检查状态机

可以重点说明：

1. 从工作列表选择到创建检查会话的过程。
2. 联锁检查如何影响是否允许曝光。
3. Ready、Exposing、Processing 等状态为什么要明确建模。

### 4.5 我如何理解 PACS 集成

可以重点说明：

1. C-ECHO 用于连通性验证。
2. C-STORE 用于发送生成结果。
3. C-FIND / C-MOVE 用于查询与回取。
4. 本地接收端为什么是回取闭环的一部分。

### 4.6 我如何理解 MWL 和 MPPS

可以重点说明：

1. MWL 负责把“待做什么检查”带到设备端。
2. MPPS 负责把“检查做到了什么状态”回传出去。
3. 本地会话状态和远端工作流状态之间需要映射关系。

## 5. 当前项目还能暴露出哪些不足

在面试里主动提到不足，通常比回避更好：

1. 还没有实现更完整的异常恢复和补发策略。
2. 还没有扩展到 MPR、多视口联动等高级阅片能力。
3. 测量系统仍是基础版，交互能力还有提升空间。
4. 当前更偏学习和演示闭环，不是完整商用系统。

## 6. 一段适合面试开场的总结

可以把这个项目概括为：

“这是一个从 WPF 客户端视角切入的医学影像学习项目。我不仅实现了本地 DICOM 导入和基础阅片，还补了曝光流程模拟、PACS 通信、MWL/MPPS 工作流、SQLite 持久化和审计日志。这个项目让我从单纯做桌面界面，过渡到理解医疗影像软件的分层架构、数据模型、标准通信和流程可追踪性。”
