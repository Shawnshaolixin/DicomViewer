# 知识地图（面试版）

## 1. 总览

如果把这个项目当成一次“从 WPF 开发转向医学影像软件开发”的学习路径，可以把知识拆成 6 条主线：

1. WPF 客户端与 MVVM/Prism
2. DICOM 数据模型
3. 图像显示基础
4. PACS 与 DICOM 网络通信
5. 医疗工作流
6. 工程化能力

这 6 条主线不是孤立的，而是围绕“从检查任务到影像回看”的完整链路串起来的。

## 2. 主链路视角

可以按下面的顺序理解整个系统：

1. 工作列表或查询结果进入系统。
2. 用户选择一条检查任务，创建本地检查会话。
3. 系统执行联锁检查，决定是否允许进入曝光流程。
4. 曝光后生成 DICOM 数据。
5. 结果可以本地回看，也可以发往 PACS。
6. 如需标准工作流，还要处理 MWL 与 MPPS。
7. 全过程需要配置、持久化、审计和测试支撑。

## 3. 六大知识模块

### 3.1 WPF / MVVM / Prism

需要掌握：

1. View、ViewModel、Command、Binding 的职责边界。
2. Prism 容器、页面导航和依赖注入。
3. 为什么界面层不应直接处理 DICOM 解析、数据库访问和网络调用。

在本项目里的价值：

1. 帮助把 UI 交互和业务逻辑解耦。
2. 帮助把查看器和曝光控制台组织成可扩展页面。

### 3.2 DICOM 数据模型

需要掌握：

1. Patient / Study / Series / Instance 的层次。
2. 常见 UID 的作用。
3. 元数据与像素数据的区别。
4. 文件格式与网络数据集之间的关系。

在本项目里的价值：

1. 决定目录扫描后如何分组。
2. 决定界面里如何展示患者、检查、序列和切片。

### 3.3 图像显示基础

需要掌握：

1. 窗宽窗位的意义。
2. 灰度映射和基础彩色图像显示。
3. 单帧、多帧、切片切换。
4. 缩放、平移、旋转、翻转与坐标变换。
5. 长度、角度测量与像素间距的关系。

在本项目里的价值：

1. 决定查看器是否具备基础可用性。
2. 决定用户看到的不只是原始像素，而是可操作的临床图像视图。

### 3.4 PACS / DICOM 网络通信

需要掌握：

1. C-ECHO、C-STORE、C-FIND、C-MOVE 的用途。
2. AE Title、Host、Port 的基本含义。
3. 查询、发送、回取三类动作的差异。
4. 本地接收端在回取流程中的作用。

在本项目里的价值：

1. 让系统从“本地查看器”升级为“可与外部影像系统交互的客户端”。
2. 让生成的结果能进入更真实的学习闭环。

### 3.5 医疗工作流

需要掌握：

1. Worklist 的作用。
2. MWL、MPPS 在标准 Modality Workflow 中的位置。
3. 会话状态、执行状态和远端状态的映射。
4. 为什么医疗流程比普通软件更强调状态一致性。

在本项目里的价值：

1. 让系统不仅知道“图像是什么”，还知道“检查进行到了哪一步”。
2. 让控制台、采集、发送、回看之间有业务上下文。

### 3.6 工程化能力

需要掌握：

1. 持久化配置与业务记录。
2. 审计日志设计。
3. 异常处理与失败反馈。
4. 自动化测试与回归验证。

在本项目里的价值：

1. 让学习项目不只是 Demo，而是具备基本工程形态。
2. 让关键流程可以验证、回放和定位问题。

## 4. 面试官常看的能力映射

### 4.1 从模块到能力

| 模块 | 常见考察点 |
| --- | --- |
| WPF / Prism | 分层意识、MVVM、交互组织能力 |
| DICOM 数据模型 | 是否理解医学影像对象结构 |
| 图像显示 | 是否理解窗宽窗位、坐标变换和测量基础 |
| PACS 通信 | 是否理解标准命令和联调要点 |
| 医疗工作流 | 是否理解 MWL、MPPS、状态机和审计 |
| 工程化 | 是否具备测试、持久化、错误处理意识 |

### 4.2 从能力到表达

面试里不要只说“我用了 fo-dicom”或“我用了 Prism”，更重要的是说清楚：

1. 为什么要这么分层。
2. 为什么要抽象接口。
3. 为什么要保留审计日志。
4. 为什么状态机要明确建模。
5. 为什么医疗软件比普通业务软件更强调一致性和可追踪性。

## 5. 建议的学习补强顺序

1. 先巩固 DICOM 基础模型和常见标签。
2. 再巩固窗宽窗位、像素映射和测量坐标基础。
3. 再理解 C-ECHO / C-STORE / C-FIND / C-MOVE 的差异。
4. 再理解 MWL / MPPS 与本地状态机的映射。
5. 最后补医疗软件中的审计、隐私保护、异常恢复和可靠性设计。

## 6. 面试准备时如何使用这份知识地图

建议同时准备三类材料：

1. 项目总结：讲“我做了什么”。
2. 知识地图：讲“我需要懂什么”。
3. 面试题库：讲“面试官可能怎么问，我怎么答”。

这样你的准备路径会从“会做项目”提升到“能结构化表达项目”。
